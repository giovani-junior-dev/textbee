# Internacionalização (i18n) + Rebranding Wablast SMS — painel `web/`

Implementação de i18n com **next-intl** (App Router) + troca de marca textbee → **Wablast SMS**
e tema de cores teal. PT-BR é o idioma padrão, com **EN disponível via toggle**.

## Arquitetura

- **Lib:** `next-intl`.
- **Sem prefixo de locale na URL** — o idioma é resolvido por **cookie** `NEXT_LOCALE`
  (default `pt-BR`). As rotas (`/dashboard`, `/login`...) não mudam.
- Arquivos:
  - `i18n/locales.ts` — locales (`pt-BR`, `en`), default, nome do cookie, labels.
  - `i18n/request.ts` — `getRequestConfig`: lê o cookie e carrega o JSON de mensagens.
  - `messages/pt-BR.json` (fonte primária) e `messages/en.json` (paridade de chaves).
  - `app/layout.tsx` — `NextIntlClientProvider` + `getLocale`/`getMessages` + `<html lang>`.
  - `components/shared/language-switcher.tsx` — grava o cookie + `router.refresh()`.
  - `global.d.ts` — tipa as mensagens (`IntlMessages`) → autocomplete e erro de build em chave inexistente.
  - `next.config.js` — envolto com `createNextIntlPlugin`. **O bloco `i18n` legado do Pages Router foi removido** (incompatível com App Router).

## Como usar nos componentes

**Client component** (`'use client'`):
```tsx
import { useTranslations } from 'next-intl'
const t = useTranslations('dashboard')
return <h2>{t('title')}</h2>
```

**Server component**:
```tsx
import { getTranslations } from 'next-intl/server'
const t = await getTranslations('footer')
```

- **Interpolação:** `t('welcome', { name })` com `"welcome": "Bem-vindo de volta, {name}"`.
- **Rich text / links:** `t.rich('s2', { link: (c) => <a href="...">{c}</a> })` com `"s2": "... <link>texto</link> ..."`.
- **Plural (ICU):** `"limitReached": "{count, plural, one {# aparelho} other {# aparelhos}}"`.
- **Validação (zod):** crie o schema **dentro** do componente para usar `t()` nas mensagens.
- **Nome da marca** ("Wablast SMS"): é constante, pode ficar como literal (não traduzido).

## Adicionar uma string nova

1. Escolha o namespace (ex.: `dashboard`, `auth`, `devices`, `apiKeys`, `messaging`, `overview`, `getStarted`, `nav`, `footer`, `common`, `apiKey`).
2. Adicione a chave em **`messages/pt-BR.json` E `messages/en.json`** (paridade obrigatória).
3. Use `t('chave')` no componente.
